# 🛡️ Insurance & Finance IT Management Platform (Frontend)

본 프로젝트는 보험 및 금융 IT 인프라의 외부 인터페이스를 실시간으로 모니터링하고 제어하기 위한 **통합 관리 플랫폼**의 프론트엔드입니다. 고도의 안정성과 추적성이 요구되는 금융 환경에 최적화된 설계 의도를 담고 있습니다.

---

## 🛠️ Technology Stack

### Core
- **React (v18/19)**: 컴포넌트 기반 아키텍처를 통한 UI 재사용성 및 상태 관리의 효율성 확보.
- **TypeScript**: 금융 데이터의 무결성을 보장하고, 개발 단계에서 런타임 오류를 사전에 방지하여 유지보수성 극대화.

### UI & Styling
- **Mantine UI (v7+)**: 금융권 어드민의 신뢰감을 주는 정교한 디자인 시스템. Tailwind CSS의 유연성과 정밀한 컴포넌트 설계를 결합하여 생산성 및 일관성 있는 UX 제공.
- **Tabler Icons**: 고해상도 환경에서도 명확한 시인성을 제공하는 벡터 아이콘 라이브러리 사용.

### Data Visualization
- **Recharts (@mantine/charts)**: 실시간 트래픽 및 장애 통계의 직관적인 관제를 위한 고성능 차트 라이브러리. 대량의 시계열 데이터를 부드럽게 렌더링.

### Communication & Security
- **Axios**: 인터셉터를 활용한 JWT 기반 보안 인증 체계 및 공통 에러 핸들링 구현.

---

## 📐 페이지별 역할 및 설계 의도

### 1. 📊 Dashboard (실시간 상태 관제)
> **설계 의도: 가시성(Visibility) 확보**
- **SOC(Security Operations Center) 스타일**의 다크 테마 레이아웃을 적용하여 실시간 지표 관제 시 눈의 피로도를 낮추고 데이터 집중도를 높였습니다.
- 주요 지표(오늘의 호출수, 성공률, 에러 건수, 평균 응답 시간)를 상단에 배치하여 시스템의 건강 상태를 0.1초 만에 파악할 수 있도록 설계했습니다.
- 실시간 이슈 타임라인을 통해 장애 발생 시 에러 원인을 즉시 노출함으로써 대응 시간을 단축시킵니다.

### 2. ⚙️ Interface 관리 및 실행 (제어 편의성)
> **설계 의도: 제어 편의성(Controllability) 강화**
- 등록된 수많은 API Config를 기관별/프로토콜별로 체계적으로 관리합니다.
- '즉시 실행' 기능을 통해 운영자가 별도의 툴 없이도 인터페이스 정상 작동 여부를 테스트하고 수동으로 프로세스를 가동할 수 있는 제어권을 제공합니다.

### 3. 🔍 Logs (트래픽 상세 분석)
> **설계 의도: 추적성(Traceability) 구현**
- 모든 인터페이스 호출에 대한 **Correlation ID** 기반 추적을 지원합니다.
- 요청(Request)과 응답(Response) 페이로드를 코드 하이라이팅이 적용된 JSON 형태로 제공하여, 개발자 및 운영자가 장애의 근본 원인(Root Cause)을 심층적으로 분석할 수 있는 환경을 구축했습니다.

## 🚀 Getting Started

### 1. 의존성 설치
```bash
npm install
```

### 2. 개발 서버 구동
```bash
npm run dev
```

### 3. 환경 설정
- `src/api/axiosInstance.ts` 파일에서 백엔드 API 서버 주소(`baseURL`)를 확인하십시오. (기본값: `http://localhost:3000`)
